Starting Python

x
 

To start Python, go to a command line or terminal. In Windows, you can find it at Start Menu > Programs > Accessories > System Tools > Command Line, or something similar, depending on your version of Windows. On Mac OS X, look in the Utilities directory in your Applications. Linux, BSD, or other operating systems, I imagine you have an idea of where to look.

x
 

When the terminal is open, type "python" and hit enter. The Python interpreter should start, telling you the version, and giving you a prompt that looks like ">>>". We're good to go!

Variables

It's been a bit annoying typing out every object or primitive. Instead we can save them in variables. Variables can store the various objects we use, with a more convenient name. They also can be very useful for generalising and performing calculations, by acting like pronumerals in algebra. That is, we can substitute them in where we've had an int, a float, a string, or other objects we're yet to see.

You can use alphanumeric characters or underscores in variable names, but there are a few good practices to follow when naming your variables.

  1. Always use a descriptive name (for example, username or output_string, compared to a or b)
  2. Don't use a capital letter for the first letter of your variable name, this signifies a different data structure.
  3. Lowercase with underscores separating words is the usual style.
  4. You can't use keywords as variable names, these are words with special meaning in Python, such as if, or for.

Another thing to know is that the value a variable points to can change, especially if you have a larger program. Sometimes you want this, sometimes you don't. This is often a source of errors.

Functions

x
 

So we've been typing in blocks of code each time we've wanted to run them. But this is repeating ourselves! And we said DON'T REPEAT YOURSELF! So what can we do to fix this? We can create an object holding our code. These are called functions. We can define (make) a function as follows.

In [65]:
 
def labwork():
    if supervisor_in_lab:
        print "Work hard!"
    else:
        print "Go to lunch"
. . .
In [66]:
 
type(labwork)
Out[66]:
function
. . .
In [67]:
 
print supervisor_in_lab
labwork
False
Out[67]:
<function __main__.labwork>
. . .
x
 

Just using "labwork" gives us a reference to the function, but doesn't run it. So how do we execute this function? Similar to how with methods (which are a special type of function), we had to use parentheses to call it, we have to do the same here.

In [68]:
 
labwork()
Go to lunch
. . .
x
 

And now it runs! But what are the parentheses for? We can provide parameters or arguments to the function, because a function that does only one thing isn't very useful, and using global state (like the variable supervisor_in_lab, available to all functions) can be confusing, especially as they can be changed by other functions. Arguments allow us to specify information or objects for the block of code in the function to use. We'll rewrite the labwork function to actually take an argument about whether the supervisor is in the lab or not.

In [69]:
 
def labwork(supervisor_in_lab):
    if supervisor_in_lab:
        print "Work hard!"
    else:
        print "Go to lunch"
. . .
x
 

So now the variable "supervisor_in_lab" refers to the value passed into the function. We can say the supervisor is in the lab:

In [70]:
 
labwork(True)
Work hard!
. . .
x
 

Or not:

In [71]:
 
labwork(False)
Go to lunch
. . .
x
 

And this is much more explicit, rather than relying on some state hidden in another variable that you can't see easily!

Scripts

x
 

So far we have typed everything we've wanted to run into the interpreter. But what if it's really long? What if we want to run it again and again? Wasn't this meant to make things easier?

To prevent having to go to all this effort, what we can do is save it as a script or program (the terminology differs depending who you talk to). To do this, you have to have a text editor to write your code it. IDLE, which comes with Python, is a good one to use, as is SublimeText (though if you're going to use it all the time, you should buy it and support the developers!). They offer extra features like syntax highlighting and automatic indenting, that makes it easier to write code. However, often you need to tell them that you are writing Python code.

The easiest way to do this is to simply save the file you're working on with a .py extension, for instance "hello.py". Then you can paste in various bits of code that we've typed out above, and save them in. You'll see that they'll be highlighted. If you save the file, you can then run it. Let's do this together.

x
 

In a file called "labwork.py":
