Loops and Iteration

While loops have a structure similar to something we've seen before... Where have we seen it before?

In conditionals! A while loop will continue to execute while the test it is passed evaluates to a value considered True.

It's important to remember that if the condition never evaluates to False, the loop will never end. Sometimes we want this, sometimes we don't. What situations would we want this behaviour? What situations do we want a guaranteed stop (at some point)?

x
 

We'd like a continual loop for 'eternal' processes, like servers that will wait for requests before doing something, then return to waiting. Remember if you want your program to end, a the condition to a while loop will need to evaluate to False at some point (or there'll need to be a way to break out of the loop, but we'll cover that later).

x
 

Loops can be used for many things. We have been doing operations to a single object or value. But this ignores the fact that many are collections of other things, for instance, lists and strings. What if we want to do the same thing to all values in a list?

Most commonly, we will use a loop to iterate through each element of a collection. How do you think we could do this, given our current knowledge?

In [1]:
 
# make a list (in this case of numbers)
numbers = [1, 2, 3, 4, 5]
 
i = 0 # create a marker to indicate what element of the list we're up to
length = 5 # set length of list to 5
 
while i < length:    # start a while loop
    print numbers[i] # print the element at position i
    i = i + 1        # increment i (go to the next element)

There are few things to notice in this block of code. First of all, we have comments, which are anything on a line after a hash (#). These can be used to document your code, to remind yourself and others of what you're doing, and why. In this case, my comments are too verbose, that is, we comment and explain each line of the code. While this is useful for you as a learner, the code also tells you what it does. You should try to note the intent of the code in your comments, or anything tricky or hard to understand at first glance (for someone used to the language).

Secondly, there's a mistake in the code, which means our code isn't generalised. Can you spot it?

We hardcode our length, which while works great for our list in the above example, also means we have to remember to change it if we change our list. Can you think of a better way to do this? A more generalised way?

We can use the 'len' function to fix this, so we don't have to remember to change the hardcoded length value. Trust me, you will forget!

x
 

Finally, the structure of our loop. In this case, we use the index of the elements to go through them all. This style of loop is often seen in older languages like C or Java, but Python has a better structure for this, where we don't need a separate counter. This provides a cleaner and DRYer experience, and provides an advantage that you don't need to keep track and increment your index value. Let's see this loop.

In [3]:
 
numbers = [1, 2, 3, 4, 5]
 
length = len(numbers)
 
for i in range(length):    # iterate through the numbers in the range 0 -> length - 1
    print numbers[i] # print the element at position i

You can see this is simpler, we don't have to do our own bookkeeping with an index value, and it's been generalised. But we can do better! We don't want to worry about indices at all, we're only concerned about the elements in the collection. This lets us write very clear and concise code, and has the added benefit of being optimized for speed in Python. So how can we do this?

In [2]:
 
numbers = [1, 2, 3, 4, 5]
 
for x in numbers:
    print x

You can see this code block is the simplest yet. No indices, no worrying about the length of the list, creating a range list or even doing lookups in the list. Clean, simple and fast! And you're much less likely to make a mistake!

The block of code executed by a loop is the same as any other piece of code. We can nest conditionals and loops inside each other, and in functions. But beware! Loops inside loops can run quite slowly, especially if the code in the inner loop is very intensive.

In [7]:
 
import time                 # bring in the time module from the standard library
def intensive_code(sec):    # create a function that we'll pretend is doing some intensive computation
    time.sleep(sec)         # taking 'sec' seconds (but will actually only sleep/pause execution)
 
num_iters = 10              # set the number of times we want the loops to run
intensity = 0.1             # set how intensive our 'computation' is, in seconds
 
for x in range(num_iters):
    for y in range(num_iters):
        print x, y
        intensive_code(intensity)

Files

Loops are good for more than just going through lists. Files often have a structure like a list, but each element is a line. But before we can read through them, we need to open it. The new way (as of Python 2.7, and with a little extra work, Python 2.6) is using "with", which will open the file for you and close it when you're done. This is the preferred way, as it ensures the file will be closed even if there is an error.

In [16]:
 
with open('firstfile.txt') as f:
    for line in f:
        print line

We read from a file! But why are there extra blank lines? There weren't any when I wrote it!

It turns out that all the characters you can see are only part of the characters in what you write. There are many special, invisible characters, but we'll only focus on two for the moment, "\n" and "\t". These characters represent a newline and a tab, respectively. If Python sees these in quotes when we print, it will insert a newline or tab at those points.

In [13]:
 
print 'Hello\tthere'
print "What\na\tnice\nday!"

But hang on! That doesn't explain why there are empty lines between the lines of our file. We can use a method of files to see what each line looks like, in a list.

In [21]:
 
with open('firstfile.txt') as f:
    print f.readlines()
['Hello!\n', 'I am\n', 'text\n', 'in \n', 'a\n', 'file!\n']
. . .
x
 

What do you notice? There are newline characters at the end of each line (each line is an element in this list). Python splits the file into lines based on newline characters, but leaves them in. The problem is that when we use print, it also adds a newline character, so we end up with two!

In [22]:
 
print "Bonsoir\n\nmes amis!"

mes amis!

So how do we get around this? There's another method of files to the rescue!

with open('firstfile.txt') as f:
    for line in f:
        print line.strip()

What does the .strip() method do? This is a good time to introduce the Python docs and standard library. No good programmer should be without the docs for their language!

The docs can be found online, but you can download them and keep a set nearby (I do!).

There's quite a bit of information here, but what we're interested in is the library reference, which documents all the different bits and pieces in the standard library of Python. But what is the standard library?

The standard library is a collection of commonly used, prewritten and optimised functions and objects for a language. Python is deemed 'batteries included', because of the breadth of it's standard library. If you're doing something, see if there isn't already code for it first!

In this case, we're interested in the strip method. But where do we find it? That depends on what it acts on... So what is line, in our above code? You can edit it and find out using 'type(line)', or if you're in IPython, 'line?'.

Hopefully you found it was a 'str' or string, either by looking at the code or introspecting it. So we need to look at the 'str' methods, in the Sequence Types section of the builtin types description. The methods are organised alphabetically, so find strip, and see if you can understand what it does.
